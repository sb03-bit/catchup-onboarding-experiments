# CatchUp 온보딩 Cold-start UX 실험 (경량판)

협업툴 연동 전 "빈 깡통" 상태에서, 사용자가 가치를 경험하기 전에 설정 부담을 먼저 지는 문제를 어떻게 풀 것인가 — 5개 설계 가설을 한 화면에서 비교하는 인터랙티브 목업입니다.

> 전체가 **단일 `index.html` 한 파일(~40KB)** 입니다. 외부 의존성·빌드 없음. 더블클릭으로 바로 열리고, GitHub Pages 업로드도 가볍습니다.

## 실험 목록

| # | 주제 | 가설 |
|---|---|---|
| 1 | 가치 미리보기 (Value-first) | 연동 전 샘플 답변·출처를 미리 보여주면 설정 동기가 유지된다 |
| 2 | 점진형 온보딩 (Progressive) | 1개만 먼저 연동시켜 작은 성공을 주면 완주율이 오른다 |
| 3 | 80% 매핑 대시보드 | 매핑 조건을 어드민이 팀원을 독려하는 진척 대시보드로 풀면 빨라진다 |
| 4 | 빈 상태 가이드 (Empty state) | 빈 홈을 '다음 할 일'로 재정의하면 이탈이 준다 |
| 5 | 모달 vs 인라인 (A/B) | 온보딩 제시 방식에 따라 완주율·이탈이 다르다 |

각 탭의 카드·버튼을 클릭하면 loading·empty·success·error 상태 전환이 실제로 동작합니다.

## GitHub Pages로 공유
1. 이 폴더를 repo에 push
2. **Settings → Pages → Source: `main` / `/root`**
3. 발급되는 `https://<계정>.github.io/<repo>/` 주소를 팀에 공유

## 제약 조건 (공통 전제)
- 최소 1개 커넥터 연동 필수
- 루트어드민 본인 계정 매핑
- 사내 사용자 80% 이상 매핑 시 임베딩(정확한 답변) 활성화
